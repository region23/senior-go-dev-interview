## Введение

Этот комплект материалов предназначен для подготовки к техническим интервью на позицию Senior-разработчика в компаниях Яндекс и Озон. Материалы учитывают специфику собеседований в этих компаниях и фокусируются на ключевых областях:

- Golang и многопоточность
- Алгоритмы и структуры данных
- Системный дизайн
- Базы данных и SQL
- Инфраструктура и мониторинг

Материалы разработаны с учетом вашего опыта работы тимлидом и необходимости освежить технические знания для возвращения в разработку.

## Советы по прохождению интервью

### Общие советы

1. **Подготовка к интервью**:
   - Изучите информацию о компании и команде
   - Повторите ключевые концепции по всем темам
   - Потренируйтесь решать задачи вслух, проговаривая ход мыслей
   - Выспитесь перед интервью

2. **Во время интервью**:
   - Внимательно слушайте вопросы, не стесняйтесь уточнять
   - Проговаривайте ход мыслей, даже если не знаете полного решения
   - Начинайте с простого решения, затем улучшайте его
   - Обсуждайте компромиссы разных подходов
   - Будьте честны, если чего-то не знаете

3. **После интервью**:
   - Запишите вопросы, которые вызвали затруднения
   - Проанализируйте свои ответы и подготовьтесь лучше к следующему интервью

### Специфика интервью в Яндексе

1. **Алгоритмические задачи**:
   - Фокус на эффективности решения
   - Анализ сложности по времени и памяти
   - Оптимизация решения

2. **Системный дизайн**:
   - Масштабируемость и отказоустойчивость
   - Распределенные системы
   - Балансировка нагрузки

3. **Технические вопросы**:
   - Глубокое понимание языка программирования
   - Многопоточность и конкурентность
   - Работа с базами данных

### Специфика интервью в Озоне

1. **Алгоритмические задачи**:
   - Практические задачи на структуры данных
   - SQL-запросы и оптимизация

2. **Архитектурные вопросы**:
   - Проектирование сервисов
   - Микросервисная архитектура
   - Интеграция систем

3. **Технические вопросы**:
   - Знание Go и его рантайма
   - Паттерны конкурентного программирования
   - Работа с базами данных и SQL
